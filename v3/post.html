Learned Weight Perturbations as a Cognitive Tool for Language Models

A language model can access thoughts outside its default reasoning manifold by temporarily perturbing its own weights. We train a 20k-parameter policy network to navigate a 16-dimensional space of weight modifications on a frozen 1M-parameter model, producing coherent alternative reasoning trajectories that the base model alone would rarely generate.

1. The problem: models are trapped in their own manifold

A trained language model occupies a specific point in weight space. That point defines a manifold of behaviors — the set of all outputs reachable through different inputs and sampling strategies. Temperature, top-p, prompting tricks: these all explore within the manifold. They cannot access behaviors that require different internal representations.

This is a fundamental constraint. When a model encounters a problem where its default associations lead nowhere, it has no mechanism to step outside its own reasoning patterns. It can rephrase, it can chain thoughts, but it cannot think differently — it cannot activate associations or reasoning pathways that its particular weight configuration doesn't support.

We propose a direct solution: give the model the ability to temporarily modify its own weights. Not randomly, and not by retraining — but through a learned policy that navigates a low-dimensional space of weight perturbations, producing coherent alternative behaviors on demand.

Core claim: A 20,768-parameter policy network can learn to navigate a 16-dimensional space of rank-1 LoRA perturbations applied to a frozen 1M-parameter language model, producing text that is measurably different from the base model (KL divergence 0.52 at full strength) while remaining coherent. The perturbations alter narrative preferences — character types, settings, emotional tones, plot structures — without destroying grammatical or logical consistency.

2. Mechanism: a policy over weight space

The system has four components. A frozen base language model. A fixed set of LoRA basis vectors that define a low-dimensional subspace of "nearby" weight configurations. A perturbation policy that learns which combinations of these basis vectors produce useful alternative behaviors. And a dream executor that orchestrates the full pipeline: read the model's state, query the policy, perturb weights, generate, revert.

┌────────────────────────┐ │ Frozen Base Model │ │ TinyStories-1M │ │ (1M params, frozen) │ └──────────┬─────────────┘ │ ┌─────────────┼─────────────┐ │ │ │ ▼ ▼ ▼ ┌─────────┐ ┌──────────┐ ┌──────────┐ │ State │ │ LoRA │ │ Dream │ │ Extract │ │ Basis │ │ Executor │ │ (16-d) │ │ (28.6k) │ │ │ └────┬─────┘ └────┬─────┘ └────┬─────┘ │ │ │ ▼ │ │ ┌──────────┐ │ │ │ Policy │───────┘ │ │ (20.7k) │─────────────────────┘ │ s → a │ └──────────┘ Total trainable: 49,440 parameters Controlling: 1,000,000 frozen parameters

The perturbation pipeline executes in five steps:

1. State extraction. During a forward pass, we extract a 16-dimensional summary of the model's cognitive state: logit entropy, top-k probability mass, hidden state statistics. This is computed at two granularities — at the last token position (what the model is about to predict) and averaged over the full sequence (the model's aggregate state). Eight features at each granularity yield a 16-dimensional state vector.

2. Policy query. The state vector feeds into a small MLP (16 → 128 → 128 → 16) that outputs a 16-dimensional action vector, squashed through tanh to [-1, 1]. Each dimension controls the coefficient of one LoRA basis direction. This network has 20,768 parameters.

3. Weight perturbation. The action vector is combined with 16 rank-1 LoRA basis directions to produce weight deltas: ΔW = α Σk ak Ak BkT. These deltas are applied in-place to the attention output projections and MLP output projections of layers 2–5 (the middle 50% of the 8-layer model). The perturbation is typically 8–15% of the original weight norm.

4. Dream generation. With weights perturbed, the model generates up to 48 tokens at temperature 0.9. This produces text that reflects the altered weight configuration — different associations, different narrative preferences, different reasoning paths.

5. Weight reversion. The deltas are subtracted, restoring the original weights exactly. The generated dream text can then be inserted into the context for the base model to process.

3. Architecture and components

Base model

We use TinyStories-1M, a GPT-Neo model with 8 transformer layers, hidden dimension 64, and vocabulary size 50,257. Trained on synthetic children's stories, it produces grammatical, coherent short narratives. All 1M parameters are permanently frozen.

The perturbation-to-weight ratio ranges from 8.5% to 14.6%. Too small (ratio < 1%), and the perturbation has no effect. Too large (ratio > 30%), and the model collapses into repetition or gibberish.

The total LoRA basis has 28,672 parameters: 16 directions × 8 modules × (Ak + Bk). These parameters are trained jointly with the policy during Phase 1. After training, they are frozen.

The final learned log-std values are approximately 0.03 across all dimensions, indicating the policy has converged to a confident, low-variance solution.

State features

The 16-dimensional state vector captures the model's cognitive state at two granularities. At the last token position (8 features):

logit entropy · top-1 probability · top-5 mass · top-20 mass · hidden mean · hidden variance · hidden norm · logit standard deviation

The same 8 features computed as a masked average over the full sequence yield the remaining 8 dimensions. These statistics are extracted during the forward pass with negligible cost — no additional model calls, no attention matrix computation, no gradient tracking.

4. Training the perturbation policy

The policy is trained via GRPO (Group Relative Policy Optimization) to discover perturbations that produce coherent but distinct outputs. Training runs for 400 steps, sampling 12 actions per step, on a fixed set of 8 story prompts.

Reward function

The reward has three components. The key innovation is the sweet-spot KL reward: instead of maximizing KL divergence (which leads to destruction) or minimizing it (which leads to no effect), we reward perturbations that produce KL near a target of 0.3:

RKL = exp(−(KL − target)² / 2σ²)

This Gaussian reward function peaks at the target KL and falls off symmetrically. Combined with a per-prompt variance bonus (encouraging the perturbation to affect different prompts differently) and a collapse penalty (fraction of positions with entropy below 0.5), the full reward is:

R = RKL + 0.3 · std(KL per prompt) − 2.0 · collapse_rate

The GRPO update uses a clipped surrogate objective with importance sampling. At each step, 12 actions are sampled from the current policy, evaluated against the reward, and the advantages are normalized within the group. The clipped ratio prevents large policy updates.

8. Implications

Weight space is navigable

The central result is that a small policy network can learn to navigate weight space productively. The 16-dimensional LoRA subspace, despite being randomly initialized, contains directions that produce coherent, semantically distinct alternative behaviors. A 20k-parameter policy trained with 400 GRPO steps can find these directions reliably.

This suggests that the local geometry of weight space around a trained model is structured: not all directions are equally useful, and the useful directions correspond to interpretable behavioral changes. The policy's convergence behavior — saturating high-impact directions while zeroing low-impact ones — indicates it is performing a form of automatic feature selection in weight space.

Perturbation magnitude matters precisely

We found a critical sensitivity to the perturbation scale α. At α = 0.04 (the original spec value, designed for Qwen-1.5B), KL was 0.0002 — effectively zero perturbation. At α = 4.0, KL was 2.3 — the model produced gibberish. The working regime was α = 2.0, giving perturbation-to-weight ratios of 8–15% and KL around 0.5.

This ratio appears to be a function of the model's size and architecture. Smaller models with smaller weight norms require proportionally larger perturbation scales. The sweet-spot KL target must be calibrated per-model: too low and nothing happens, too high and coherence collapses.

The system is remarkably cheap to train: 49,440 parameters, 25 seconds on a single GPU, 400 GRPO steps. The inference cost of a dream is exactly one forward pass for state extraction, one MLP forward pass for the policy (negligible), and one generation call with modified weights (identical cost to normal generation). Weight modification is in-place addition and subtraction — O(|ΔW|) which is far smaller than a forward pass.

The dream as a cognitive operation

The dream executor implements a complete cognitive operation: assess the current state, decide how to perturb, generate under altered weights, revert, and return the result. The dream text can be inserted into the model's context, giving it access to thoughts it could not have produced with its default weights.

This is qualitatively different from sampling with different temperatures or using different prompts. Those approaches explore within the model's existing behavioral manifold. Weight perturbation accesses a different manifold — a nearby model that has different associations, different narrative preferences, different default behaviors. The text generated by this nearby model is then available to the original model through the context window.

Whether this produces genuinely useful reasoning — whether the model can leverage dream text to solve problems it otherwise couldn't — is the subject of Phase 2 (meta-adapter training), which learns when to invoke dreams during task completion. Our Phase 1 results establish the prerequisite: that the perturbation policy can reliably produce coherent, meaningfully different alternative text. Phase 2 trains a LoRA adapter (rank 8 on q/v projections, 16k parameters) to decide when this capability is beneficial.

Limitations

TinyStories-1M is a toy model. The stories it produces are simple, and the behavioral shifts — while measurable and consistent — are correspondingly simple: character swaps, setting changes, tone shifts. On a larger model with richer internal representations, we would expect the perturbation directions to align with more complex behavioral axes: reasoning strategies, domain expertise, creativity vs. precision.

The state representation (16 scalar statistics) is crude. A richer state — perhaps including attention patterns or intermediate layer representations — would allow the policy to make more context-sensitive perturbation decisions. Currently, the policy produces nearly identical actions for all our test prompts because their 16-dimensional state vectors are similar.

The LoRA basis is randomly initialized and fixed during training. A learned basis — perhaps optimized to maximize the diversity of achievable behaviors — could produce a richer and more efficiently navigable perturbation space.

What this points toward

If a 1M-parameter model can benefit from temporarily stepping into nearby weight configurations, larger models should benefit more. The weight space of a 70B-parameter model contains far richer structure. The relevant question is not whether useful perturbation directions exist — they almost certainly do — but whether a small policy can find them efficiently.

The dream operation could become a standard cognitive tool: when the model is uncertain, when its default associations lead nowhere, when the problem requires a perspective shift — it temporarily becomes a slightly different model, generates alternative thoughts, and imports them back through the context window. The base model's weights never change. Only the policy and basis need to be trained. The cost is one additional generation step.

Summary: We demonstrate that a learned policy over weight perturbations can produce coherent alternative reasoning from a frozen language model. The system — 49k trainable parameters, 845 lines of code, 25 seconds to train — reliably generates text that is measurably different from the base model (KL = 0.52) while maintaining coherence (collapse rate 0.3%). Individual perturbation directions correspond to interpretable behavioral axes. The perturbation strength is a smooth, predictable dial. The mechanism generalizes to unseen prompts. This establishes the prerequisites for using weight perturbation as a cognitive tool: the ability to step outside the model's default behavioral manifold and return with alternative thoughts.
