What Happens When You Randomly Poke a Language Model's Weights

We found unnamed behavioral knobs inside a frozen language model. Each one smoothly controls a different aspect of the model's personality. None of them were trained. They were just there.

Take a frozen language model. Don't fine-tune it. Instead, pick a random direction in weight space—literally torch.randn—and nudge the weights along it. What happens?

You'd expect the model to break. Instead, it changes personality.

And different random directions change it in different ways. One direction controls how abstract the writing is. Another controls structural complexity. A third controls emotional tone. We didn't train any of this. We just pointed in random directions in a 33-million-dimensional space and found that each direction has a name.

Here's what we found.

These effects don't appear at all scales. There's a sweet spot, and on either side of it, nothing interesting happens.

The existence of this band is not obvious. The model could have been brittle. It isn't. Behavior is a smooth function of weights, and there's room to explore.

This is nested control—the fine knob's effect depends on the coarse setting. The coarse knob defines a regime, and the fine knob operates within it.

This is the kind of structure you'd expect from a well-designed control system. We didn't design it. It just fell out of random low-rank weight edits.

How It Works

The technical setup is simple. We freeze TinyStories-33M and define 16 rank-1 LoRA basis directions on the attention output projection and MLP output projection of each layer. Each basis direction is a pair of random vectors producing a rank-1 weight edit. A 16-dimensional action vector controls how much of each edit to apply.

action a ∈ R¹&sup6; → Δw = Σ_k a_k · (A_k ⊗ B_k) → w' = w + Δw → generate

We wrap this in a model-based RL loop. An actor (Gaussian policy, trained with GRPO) learns to sample interesting action vectors. A world model (small transformer) learns to predict the behavioral consequences of an action without running the real model. After enough real rollouts, the actor can "dream"—exploring weight space through the world model's predictions.

Training dynamics
Phase 1 (real rollouts):  reward +2.88 → +3.18  |  world model MSE: 2.18 → 0.07
Phase 2 (dreaming):      dream reward: +4.46    |  real validation: +3.11

The reward function doesn't know what a "good" behavioral change looks like. It rewards structure: high KL divergence (the edit did something), low KL variance across prompts (it did the same thing to every prompt), and maintained coherence. The actor learns to find consistent, global behavioral shifts rather than prompt-specific noise.

What This Means

Linear probing showed us that representations are linearly structured. Steering vectors showed us that activations vary linearly. This is the weight-space version of the same story: behavior varies smoothly in weight space, and low-rank edits are enough to access that variation.

But the RL framing adds something new. We're not searching for specific effects. We're doing unsupervised behavioral cartography—discovering what kinds of changes are possible, without knowing what we're looking for. The actor explores, the world model maps the territory, and what falls out are named, interpretable behavioral axes that nobody put there.

This is v1, run on a 33M-parameter toy model. The random basis works, but a trained basis should find sharper axes. The world model works, but it's optimistic (dream reward overshoots real reward by ~30%). The hierarchy is real, but it's faint at this scale.

The question is what happens at 7B parameters, with rank-8 edits instead of rank-1, and a world model that's had time to learn the real landscape. If random directions in a 33M model already have names, what's hiding in the weight space of a model that's 200x larger?

Code: ~700 lines of pure PyTorch + transformers. No RL libraries. The environment, actor, world model, and training loop each fit in a single file.
